Flowchart

核心流程：從文件入庫到報表匯出，每一步都有資料表與審核狀態承接。

1. 文件上傳PDF / 圖片 / Excel / CSV 進入 documents，建立 OCR job 與 review task。
2. OCR 抽取分類文件類型，抽欄位、表格、金額與統編，保存 confidence。
3. 勾稽驗證401 稅額、資產負債平衡、跨報表差異與期別衝突。
4. 人工複核低信心與異常欄位進入待辦，人工修正後再入帳。
5. 會計核心科目、分錄、總帳、試算、應收應付、銀行與固定資產。
6. 稅報輸出401 草稿、財報列、月結包、CSV / 交換檔、稽核軌跡。

OCR Intelligence

OCR 不是直接改帳，而是建立可複核的驗證底稿。

9 步 OCR 流程

前處理旋轉、去噪、裁切、表格線、頁面分割提升掃描檔穩定性
分類與辨識401、財報、銀行、發票、固定資產表文字、表格、欄列關係
抽取與標準化民國年、千分位、負數、統編、科目寫入 confidence 與 raw value
驗證與複核平衡、合計、稅額試算、跨報表比對低信心欄位送人工確認

第一波文件類型

  • 資產負債表：資產、負債、權益、比較期與平衡規則。
  • 401 營業稅：銷售額、銷項稅、進項稅、不得扣抵、留抵與應納稅額。
  • 損益表：收入、成本、費用、營業外項目、稅前與稅後淨利。
  • 現金流量表：營業、投資、籌資現金流與期初期末現金。
  • 銀行、應收、應付、庫存、固定資產、年度申報附表。
OCR specification
重點是可信鏈路：OCR 先作底稿與驗證，不直接改帳。這能降低 AI/OCR 誤判造成的會計風險。

Tax Compliance

401 報表是 MVP 的合規錨點：銷項、進項、扣抵、留抵與匯出都要能追溯。

401 核心欄位與驗證

  • 所屬年月、統編、稅籍、營業人與申報類型。
  • 應稅 / 零稅率銷售額、銷項稅額、進項稅額。
  • 進貨費用、固定資產、不得扣抵、可扣抵進項。
  • 應納稅額、留抵、退稅、附件份數與申報日期。
  • 銷項稅額接近銷售額 5%，並與發票、損益表互相勾稽。
VAT return workflow
401 是很好的商業切入點：有明確表單、明確欄位、明確驗證規則，也能自然連到發票、總帳與月結鎖帳。
